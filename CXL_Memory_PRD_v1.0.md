## 1. 文档概述

### 1.1 目的
本文档定义了CXL (Compute Express Link) Type 3 内存扩展设备的完整产品需求，涵盖硬件规格、软件接口、性能指标、可靠性要求及测试验证标准。

### 1.2 适用范围
- **目标设备**: CXL Type 3 内存扩展卡/模块
- **协议版本**: CXL 2.0 / CXL 3.0 / CXL 3.1 / CXL 3.2
- **支持平台**: Linux Kernel 5.18+, UEFI 2.10+, ACPI 6.5+
- **应用场景**: 数据中心内存扩展、内存池化、分层存储、AI/ML训练

## 2. 产品定义

### 2.1 产品定位
CXL Type 3 内存设备是一种基于PCIe物理层的高性能内存扩展解决方案，支持：
- **内存容量扩展**: 为服务器提供额外的DDR内存池
- **内存池化共享**: 多台主机共享访问同一内存资源
- **内存分层**: 与系统DRAM配合实现自动分层存储
- **持久内存**: 支持非易失性内存配置

### 2.2 设备类型定义
| 类型 | 描述 | 本设备支持 |
|------|------|------------|
| Type 1 | 缓存设备 (CXL.cache) | ❌ |
| Type 2 | 加速器内存 (CXL.cache + CXL.mem) | ❌ |
| **Type 3** | **内存扩展设备 (CXL.mem)** | **✅** |

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 内存容量支持 (FR-001)
**需求描述**: 设备应支持可配置的内存容量配置

| 参数 | 要求 |
|------|------|
| 单设备最小容量 | 128 GB |
| 单设备最大容量 | 2 TB |
| 容量粒度 | 64 GB 递增 |
| 内存类型 | DDR5 RDIMM/LRDIMM |

#### 3.1.2 内存分区 (FR-002)
**需求描述**: 支持将设备内存划分为易失性和持久性区域

- **分区要求**:
  - 易失性区域必须从DPA 0开始
  - 支持纯易失性配置
  - 支持纯持久性配置
  - 支持混合配置（易失性+持久性）
  
- **配置接口**:
  - 通过Mailbox命令 `Get Partition Info` 查询分区
  - 通过 `Set Partition Info` 配置分区（如支持）

#### 3.1.3 CXL协议支持 (FR-003)
**需求描述**: 支持完整的CXL.mem协议功能

| 功能 | CXL 2.0 | CXL 3.0 | CXL 3.1 |
|------|---------|---------|---------|
| MLD (Multi-Logical Device) | ✅ | ✅ | ✅ |
| 内存池化 | ✅ | ✅ | ✅ |
| Global Persistent Flush | - | ✅ | ✅ |
| IDE (Integrity & Data Encryption) | - | ✅ | ✅ |
| 多级交换 | - | ✅ | ✅ |
| 256B Flit | - | ✅ | ✅ |
| Back Invalidate (HDM-DB) | - | ✅ | ✅ |

#### 3.1.4 HDM解码器支持 (FR-004)
**需求描述**: 设备必须实现HDM (Host-Managed Device Memory) 解码器

- **解码器数量**: 至少4个HDM解码器
- **配置能力**: 
  - 支持通过CFMWS (CXL Fixed Memory Window Structure) 编程
  - 支持"Lock on Commit"功能
  - 支持动态重新配置（非锁定状态下）
- **地址映射**: 支持HPA到DPA的地址转换

### 3.2 管理接口

#### 3.2.1 Mailbox接口 (FR-005)
**需求描述**: 实现CXL规范定义的所有必需Mailbox命令

**必需命令**:
| 命令 | 命令码 | 描述 |
|------|--------|------|
| Identify Memory Device | 0x0000 | 设备识别信息 |
| Get Partition Info | 0x0100 | 获取分区信息 |
| Get Health Info | 0x4200 | 获取健康状态 |
| Get Event Records | 0x0101 | 获取事件日志 |
| Clear Event Records | 0x0102 | 清除事件日志 |
| Get Event Interrupt Policy | 0x0103 | 获取中断策略 |
| Set Event Interrupt Policy | 0x0104 | 设置中断策略 |
| Get Timestamp | 0x0300 | 获取设备时间戳 |
| Set Timestamp | 0x0301 | 设置设备时间戳 |
| Get Security State | 0x0200 | 获取安全状态 |
| Get Shutdown State | 0x0204 | 获取关机状态 |
| Set Shutdown State | 0x0205 | 设置关机状态 |

**可选命令**:
| 命令 | 命令码 | 描述 |
|------|--------|------|
| Set Partition Info | 0x0105 | 动态分区配置 |
| Get Poison List | 0x4300 | 获取毒化地址列表 |
| Inject/Clear Poison | 0x4301/0x4302 | 毒化地址管理 |

#### 3.2.2 事件日志 (FR-006)
**需求描述**: 设备必须维护四级事件日志系统

| 级别 | 用途 | 最小条目数 |
|------|------|------------|
| Informational | 常规信息 | 1 |
| Warning | 警告事件 | 1 |
| Failure | 故障事件 | 1 |
| Fatal | 致命错误 | 1 |

**通知机制**:
- 支持MSI/MSI-X中断（OS First）
- 支持VDM (Vendor Defined Message)（FW First）

### 3.3 安全功能

#### 3.3.1 安全启动 (FR-007)
**需求描述**: 支持硬件安全启动验证

- **硬件RoT (Root of Trust)**: 支持硬件信任根
- **固件签名验证**: 验证固件镜像的数字签名
- **安全启动状态**: 支持查询和报告安全启动状态

#### 3.3.2 内存加密 (FR-008)
**需求描述**: 支持CXL IDE (Integrity and Data Encryption)

- **IDE类型**: 支持SPDM over DOE进行密钥协商
- **加密算法**: AES-GCM-256
- **密钥刷新**: 支持密钥定期轮换

#### 3.3.3 持久内存安全 (FR-009)
**需求描述**: 支持持久内存区域的安全锁定

- **安全状态**: Locked / Unlocked
- **解锁机制**: 支持通过Mailbox命令解锁
- **缓存管理**: 状态转换时触发CPU缓存刷新

## 4. 性能需求

### 4.1 带宽规格

#### 4.1.1 链路带宽 (PR-001)
**需求描述**: 支持多种链路配置

| 链路配置 | 理论带宽 (单向) | 有效带宽目标 |
|----------|-----------------|--------------|
| x4 (CXL 2.0) | 16 GB/s | 14 GB/s |
| x8 (CXL 2.0) | 32 GB/s | 28 GB/s |
| x16 (CXL 2.0) | 64 GB/s | 56 GB/s |
| x4 (CXL 3.0) | 32 GB/s | 28 GB/s |
| x8 (CXL 3.0) | 64 GB/s | 56 GB/s |
| x16 (CXL 3.0) | 128 GB/s | 112 GB/s |

#### 4.1.2 内存带宽 (PR-002)
**需求描述**: 设备内存子系统带宽

| 配置 | 读取带宽 | 写入带宽 |
|------|----------|----------|
| 1DPC (单通道) | ≥ 40 GB/s | ≥ 35 GB/s |
| 2DPC (双通道) | ≥ 80 GB/s | ≥ 70 GB/s |
| 4DPC (四通道) | ≥ 160 GB/s | ≥ 140 GB/s |

### 4.2 延迟要求

#### 4.2.1 访问延迟 (PR-003)
**需求描述**: 从主机到设备内存的访问延迟

| 场景 | 目标延迟 |
|------|----------|
| 本地访问 (缓存命中) | < 100 ns |
| 缓存未命中 (直接访问) | 200-400 ns |
| 跨NUMA访问 | 400-800 ns |

#### 4.2.2 Mailbox延迟 (PR-004)
**需求描述**: Mailbox命令响应时间

| 命令类型 | 最大响应时间 |
|----------|--------------|
| 读命令 (如Get Health Info) | < 1 ms |
| 写命令 (如Set Timestamp) | < 10 ms |
| 配置命令 (如Set Partition) | < 100 ms |

### 4.3 QoS支持

#### 4.3.1 QoS遥测 (PR-005)
**需求描述**: 支持CXL QoS遥测功能（CXL 3.0+）

- **带宽利用率**: 报告每个HDM的带宽使用情况
- **延迟统计**: 提供访问延迟百分位数据 (P99, P95)
- **拥塞监控**: 检测和报告拥塞事件

## 5. 可靠性、可用性、可服务性 (RAS)

### 5.1 错误检测与纠正

#### 5.1.1 内存ECC (RAS-001)
**需求描述**: 内存子系统必须支持完整的ECC保护

| 功能 | 要求 |
|------|------|
| ECC类型 | On-Die ECC + Side-band ECC |
| 纠正能力 | 单符号纠错 (SEC) |
| 检测能力 | 双符号检测 (DED) |
| Patrol Scrubbing | 支持 |
| Demand Scrubbing | 支持 |

#### 5.1.2 链路纠错 (RAS-002)
**需求描述**: CXL链路层错误保护

- CRC保护: 每个Flit包含CRC校验
- 重传机制: 支持链路层重传 (LLR)
- 错误计数: 维护链路错误计数器

### 5.2 毒化 (Poison) 支持

#### 5.2.1 Poison管理 (RAS-003)
**需求描述**: 支持内存毒化机制

- **毒化注入**: 支持通过Mailbox注入毒化地址
- **毒化清除**: 支持清除毒化状态
- **毒化列表**: 支持查询当前毒化地址列表
- **传播控制**: 控制毒化数据的传播行为

### 5.3 后封装修复 (PPR)

#### 5.3.1 PPR支持 (RAS-004)
**需求描述**: 支持内存行修复（CXL 3.2+）

- **sPPR (Soft PPR)**: 软件触发的行修复
- **hPPR (Hardware PPR)**: 启动时硬件自动修复
- **修复记录**: 跟踪已修复的内存行

### 5.4 健康监控

#### 5.4.1 SMART监控 (RAS-005)
**需求描述**: 提供类SMART设备健康信息

| 指标 | 描述 |
|------|------|
| Life Used | 已使用寿命百分比 |
| Device Temperature | 设备温度（摄氏度） |
| Dirty Shutdowns | 异常关机次数（持久内存） |
| Corrected Errors | 已纠正的ECC错误数 |
| Uncorrected Errors | 不可纠正的错误数 |

## 6. 软件接口需求

### 6.1 操作系统支持

#### 6.1.1 Linux支持 (SW-001)
**需求描述**: 完整支持Linux内核CXL子系统

| 内核版本 | 支持级别 | 说明 |
|----------|----------|------|
| 5.18+ | 基本支持 | CXL内存设备识别、基本访问 |
| 6.2+ | 完整支持 | HDM解码器、分区、RAS |
| 6.6+ | 增强支持 | NUMA感知、内存分层 |
| 6.8+ | 高级特性 | QoS遥测、CXL 3.x特性 |

#### 6.1.2 固件接口 (SW-002)
**需求描述**: 支持系统固件和UEFI接口

- **CEDT (CXL Early Discovery Table)**: 向系统固件报告
- **CHBCR (CXL Host Bridge Configuration Register)**: 支持MMIO访问
- **CFMWS (CXL Fixed Memory Window Structure)**: 支持固定内存窗口

### 6.2 管理工具

#### 6.2.1 cxl-cli支持 (SW-003)
**需求描述**: 兼容Linux cxl-cli工具

**支持命令**:
```bash
cxl list                    # 列出所有CXL设备
cxl identify <device>       # 获取设备信息
cxl get-health <device>     # 获取健康状态
cxl get-event-log <device>  # 获取事件日志
cxl update-firmware <device> # 固件更新
cxl poison <device>         # 毒化管理
cxl qos <device>            # QoS遥测（CXL 3.0+）
```

### 6.3 内存管理

#### 6.3.1 NUMA支持 (SW-004)
**需求描述**: 操作系统正确识别NUMA拓扑

- NUMA节点自动创建
- 内存策略支持 (libnuma)
- 自动NUMA平衡

#### 6.3.2 内存分层 (SW-005)
**需求描述**: 支持Linux内存分层

- demote/kdamond自动页面迁移
- 基于访问频率的分层策略
- 与系统DRAM协同工作

## 7. 接口规格

### 7.1 物理接口

#### 7.1.1 PCIe连接器 (HW-001)
**需求描述**: 标准PCIe接口规格

| 规格 | 要求 |
|------|------|
| 金手指 | PCIe CEM标准 |
| 链路宽度 | x4 / x8 / x16 自适应 |
| 连接器类型 | MCIO / SlimSAS / 标准PCIe |
| 卡尺寸 | HHHL (标准半高半长) 或 FHHL |

#### 7.1.2 电源规格 (HW-002)
**需求描述**: 设备功耗管理

| 参数 | 要求 |
|------|------|
| PCIe供电 | 12V / 3.3V / 3.3Vaux |
| 最大功耗 | 75W (x8), 150W (x16) |
| 支持辅助供电 | 6-pin / 8-pin PCIe电源接口 |
| 热设计功耗 (TDP) | < 25W (控制器) + 内存功耗 |

#### 7.1.3 散热要求 (HW-003)
**需求描述**: 设备散热设计

- **被动散热**: 支持自然对流散热（低功耗配置）
- **主动散热**: 支持风扇/风道设计（高功耗配置）
- **工作温度**: 0°C ~ 55°C (环境温度)
- **存储温度**: -40°C ~ 85°C

### 7.2 环境要求

#### 7.2.1 工作环境 (HW-004)

| 参数 | 要求 |
|------|------|
| 工作温度 | 0°C ~ 55°C |
| 相对湿度 | 8% ~ 90% (无凝结) |
| 海拔高度 | 0 ~ 3000米 |
| 振动 | 随机振动 0.5G RMS |
| 冲击 | 半正弦波 10G |
