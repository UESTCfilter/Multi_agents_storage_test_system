# AI Storage Test System - 模板与任务控制功能

## 更新内容 (2026-03-27)

### 1. 前端功能更新

#### 1.1 模板选择弹窗
- 点击"生成策略"/"生成设计"/"生成用例"按钮后弹出模板选择窗口
- 支持选择/不选择模板生成

#### 1.2 三种模板类型

**测试策略模板**:
- `standard` - 标准模板（通用测试策略框架）
- `enterprise` - 企业级模板（包含RAS/安全/合规要求）
- `cxl_pcm` - CXL+PCM专用模板

**测试设计模板**:
- `standard` - 标准设计模板（功能/性能/可靠性三维度）
- `detailed` - 详细设计模板（包含环境/步骤/判定标准）
- `minimal` - 精简模板（核心测试项快速覆盖）

**测试用例模板**:
- `standard` - 标准用例模板（ID/名称/步骤/结果/优先级）
- `excel` - Excel导入模板（适合批量导入测试管理工具）
- `automated` - 自动化模板（可直接转为自动化脚本）

#### 1.3 任务终止功能
- 生成任务运行时显示"终止任务"按钮
- 点击后确认终止，任务立即停止

#### 1.4 依赖控制
- 测试设计按钮：未生成策略时禁用，显示"⚠️ 需先完成测试策略"
- 测试用例按钮：未生成设计时禁用，显示"⚠️ 需先完成测试设计"

#### 1.5 状态概览
- 概览页显示三个阶段的完成状态
- 已完成显示 ✅，可生成显示 ⏳，被阻塞显示 ⏸️

### 2. 后端API更新

#### 2.1 生成接口更新

```
POST /api/projects/{id}/generate-strategy
POST /api/projects/{id}/generate-design
POST /api/projects/{id}/generate-cases
```

**请求体**:
```json
{
  "template": "standard",
  "use_template": true
}
```

#### 2.2 任务终止接口

```
POST /api/workflow/stop/{project_id}
```

**响应**:
```json
{
  "message": "任务已终止",
  "project_id": 1
}
```

#### 2.3 工作流状态接口更新

```
GET /api/workflow/status/{project_id}
```

**响应**:
```json
{
  "status": "running",
  "current_stage": "strategy",
  "progress": 45,
  "message": "使用模板: standard",
  "task_id": "1",
  "can_stop": true
}
```

### 3. 依赖关系验证

```
需求文档上传
    ↓
生成测试策略 ✓
    ↓
生成测试设计（需策略完成）✓
    ↓
生成测试用例（需设计完成）✓
```

---

### 4. 使用流程

1. **上传需求文档**
2. **点击"生成策略"** → 选择模板 → 开始生成
3. **等待策略完成**（或点击终止）
4. **点击"生成设计"** → 选择模板 → 开始生成
5. **等待设计完成**（或点击终止）
6. **点击"生成用例"** → 选择模板 → 开始生成
7. **查看/导出结果**
